## 二、灰色地带（仅记录，待你定夺）

### 1. 交易列表按顶级分类筛选不下钻子分类（建议优先定夺）
- 位置：`lib/pages/transactions_pages.dart:368-370`
- 现状：分类筛选是**精确匹配**（选顶级不带出子分类交易），而报表侧用 `rootIdOf` 把子分类**归总到顶层**。同一"顶层归总"概念两处不一致；`category_tree.dart` 已有现成 `descendantIds` 但筛选处未用。
- 待定：若"选顶级=顶级及全部子分类"是意图 → 改用 `descendantIds` 展开；若筛选面板本就平铺可直接选子分类 → 属可接受设计。**你定方向我再改。**

### 2. 导入合法但非本应用的 JSON 会静默覆盖为初始态
- 位置：`lib/app/veri_fin_controller.dart:importDataJson`
- 现状：空文件/非法 JSON 能拦截（不清空）✓；但导入"合法 JSON 却非本应用备份"（如 `{"foo":1}`）时会解析为空、把数据覆盖成默认账本+空交易，等同静默初始化。与"错误格式文件不应清空数据"的严格解读有出入。属长期宽松容错，非本轮引入。
- 待定：是否加 `app == 'verifin'` 或"至少含一个已知键"的校验，让非本应用 JSON 直接报错不清空。**改动小、可测，但触及数据关键的导入路径，按你的保守原则留你拍板。**

### 3. 首页剩余额度超支时夹到 0 显示（非回归，确认即可）
- 位置：`lib/pages/home_page.dart:445`（`(budget-expense).clamp(0,budget)`）
- 现状：超支时首页"剩余"显示 0（进度环仍变红），预算页显示负值"已超支"。刻意取舍。附带小瑕疵：`_BudgetTrendPainter` 画 4 条网格线而 Y 轴 3 个标签，最上标签与线不完全对齐（`budget_pages.dart:796-803` vs `907-917`），断言只要求"有纵轴刻度"，已满足。
